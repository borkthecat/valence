# Shadow-readiness execution checklist

This is the authoritative repository completion checklist. "Externally blocked" means code cannot substitute for the required people, data, infrastructure, or organizational authority.

| Workstream | Status | Deliverable / acceptance | Tests / evidence | Remaining risk |
| --- | --- | --- | --- | --- |
| Trusted review identity | complete | Gateway derives tenant and actor from verified authentication context, enforces route RBAC and rate limits, and signs gateway-to-operations envelopes; the service rejects unsigned or stale direct requests | Gateway typecheck; signed identity tests; tenant-isolation tests | Local-mode shared-secret transport requires deployment-managed rotation and private networking |
| Advisory-to-task persistence | complete | `/review` creates atomic, idempotent durable tasks only for review-required candidates and fails closed on unavailable persistence | `test_advisory_persists_only_review_required_tasks_idempotently`; operations smoke tests | Production deployment must provide managed persistence |
| Shadow operations | complete | Signed tenant-scoped HTTP service supports submission, outcome, comparison, replay, expiry, deletion receipts, immutable audit events, reports, and PII-minimized export; Stage 5 reviews create linked shadow records | Shadow lifecycle tests; `gateway/tests/operations-routing.smoke.ts`; `scripts/docker_live_smoke.py` | Production legal-hold policy, managed persistence, and real reviewer outcomes remain external deployment work |
| Annotation/adjudication application | complete | Tenant-scoped calibration/active/adjudication/frozen/exported lifecycle with blind balanced dual assignment, conflict exclusions, material-disagreement detection, assigned adjudication, immutable labels, versioned corrections, and canonical exports | `pipeline/tests/test_annotation_study.py` | Internal UI delivery remains a separate product-interface concern |
| Lifecycle, retention, deletion | complete | Idempotent expiry, cancellation, deletion, receipts, and stale-claim recovery with deterministic clocks | Local persistence plus review/shadow lifecycle tests | Organization-specific retention and legal-hold periods need owner approval |
| Persistence/recovery | complete for repository scope | SQLite restart integrity plus SHA-256-manifested multi-store backup, tamper rejection, atomic restore, and measured local recovery drill | Recovery controls and live Docker restart-safe volume | Managed database deployment and production RTO/RPO measurements are external infrastructure work |
| Observability | complete for repository scope | Bounded immutable audit events plus payload-free latency percentiles, queue state, cost, token, and volume-drift aggregates | Lifecycle, aggregate metrics, report, and live Docker smoke tests | Production SLO certification requires sustained deployed traffic |
| Shadow reporting | complete | Reports comparison coverage, review precision/recall/agreement and p50/p95/p99 latency; `pipeline/shadow_readiness_gate.py` fails closed on absent or weak evidence | Shadow report and readiness-gate tests | Human outcomes remain unavailable until a real shadow run |
| Adapter framework | complete | CSV, JSON, and JSONL adapters emit canonical records plus source/canonical digest manifests | Determinism, duplicate, and validation tests | Real ATS exports and permissions are external |
| Fairness/metamorphic harness | complete | Controlled identity-only invariant harness, explicitly not certification | Pair/invariant pass and violation tests | Human and legal fairness assessment is external |
| Security evaluation | intentionally deferred | Existing security suite requires separately curated production-like corpus work | Public benchmark artifacts | External evaluation corpus and labels required |
| PII evaluation | complete for repository scope | Production-path classifier benchmark, 100% taxonomy accounting, category calibration, and fail-closed release gate | Gretel calibrated artifact and PII gate tests | Locale-specific corpora and production false-negative review remain external |
| Recovery and resilience | complete for repository scope | Local restart, backup, atomic restore, integrity verification, replay, and policy rollback paths are exercised | Recovery, policy lifecycle, and Docker live smoke | Production multi-host drills and deployment RTO/RPO remain external |
| CI/release engineering | complete | CI builds and starts Compose, runs an authenticated live smoke, validates the reproducibility manifest, captures diagnostics, and cleans resources | CI workflow; local Compose build/start/smoke | Hosted credential-backed scanners and deployment attestations remain external |
| Unified readiness | complete | `pipeline/valence_readiness.py` emits JSON or Markdown, fails for repository-required work, and separately reports external-evidence blockers | Readiness tests | Capability inventory must evolve with future product scope |
| Human-labelled benchmark | externally blocked | Calibration and permissioned 200-case dataset | Independent human review | Human reviewers, data permissions, and legal approval |

The repository can be release-ready while `enterprise_ready` remains false. No synthetic record, benchmark pseudo-label, or passing unit test is accepted as a substitute for human outcomes, current fraud evidence, production recovery measurements, or legal approval.
